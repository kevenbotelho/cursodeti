Lógica de Programação - 01

ALGORITMO

Um algoritmo é uma sequência finita de instruções bem definidas que levam à solução de um problema específico. É uma descrição precisa e ordenada de como realizar ou resolver uma tarefa. Um algoritmo pode ser implementado em qualquer linguagem de programação e é independente de uma linguagem específica. Os algoritmos são extremamente necessários para a programação, pois são a base para a implementação de soluções em formato de código.

Funcionamento

Quando um algoritmo é executado, cada passo é seguido exatamente na ordem especificada até que o objetivo seja alcançado ou uma condição de parada seja encontrada. Eles são independentes da linguagem de programação específica e podem ser implementados em diferentes ambientes.

Lógica de Programação

É como pensamos para resolver um problema. Imagine isso como seguir uma receita de bolo. Você precisa entender cada passo e organizá-los de forma lógica para que o bolo saia certo no final. Isso envolve saber usar os ingredientes (dados) da maneira certa e seguir a ordem correta dos passos.

Variável

Imagine uma variável como uma caixa onde você pode guardar alguma coisa, como um número ou uma palavra. Você pode dar um nome para essa caixa e colocar diferentes coisas dentro dela enquanto o programa está sendo executado. Essas "caixas" são úteis para armazenar informações temporariamente e usar mais tarde no programa.

Pseudocódigo

Escrever um algoritmo é como dar instruções para resolver um problema usando uma linguagem simples que todos entendam, sem se preocupar com as regras de programação. É como fazer um esboço de um plano antes de começar a fazer algo. Você descreve passo a passo o que precisa ser feito de forma clara e direta, ajudando a entender como resolver o problema antes de escrever o código real em uma linguagem de programação específica. Os algoritmos que vamos trabalhar serão feitos em pseudocódigo.

Regras para Criar um Algoritmo

  • Usar somente um verbo por frase (Um comando)
  • O objetivo precisa ser claro
  • Usar frases curtas e simples
  • Não utilizar palavras com duplo sentido

Componentes Básicos de um algoritmo

  • Dados de entrada: São os dados que o usuário vai digitar. São os dados necessários para que o programa seja executado. São variáveis.
  • Dados de processamento: São aqueles que o algoritmo "manipula" ou "trabalha" durante sua execução para produzir o resultado desejado.
  • Dados de saída: São os resultados ou informações que o algoritmo produz após processar os dados de entrada. Eles representam a resposta ou o resultado final do algoritmo com base nas operações realizadas.

    • Exemplo - MEDIA

  • Entrada: Escolha três números.
  • Processamento: Some os três números.
    Divida a soma pelo número total de números (que é 3 neste caso).
  • Saída: O resultado do cálculo é a média dos três números.

Tipos de Algoritmos

    DESCRIÇÃO NARRATIVA

    Utiliza linguagem do dia a dia para dar instruções

    Vantagem: Facilidade ao criar.

    Desvantagem: Pode ter duplo sentido.

    EXEMPLO:

      Calcular a média

    • Solicite ao usuário 3 valores separados por vírgulas.
    • Calcular a média dos 3 valores fornecidos.
    • Mostrar o resultado da média na tela.

    Neste exemplo, a ambiguidade se encontra na instrução "3 valores separados por vírgulas". Não fica claro se o usuário deve inserir três valores separados por vírgulas em uma única entrada ou se ele deve fornecer três entradas separadas. Isso poderia levar a diferentes interpretações ao executar o programa.
