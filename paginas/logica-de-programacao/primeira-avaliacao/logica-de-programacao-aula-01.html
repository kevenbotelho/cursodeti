Lógica de Programação - 01

ALGORITMO

Um algoritmo é uma sequência finita de instruções bem definidas que levam à solução de um problema específico. É uma descrição precisa e ordenada de como realizar ou resolver uma tarefa. Um algoritmo pode ser implementado em qualquer linguagem de programação e é independente de uma linguagem específica.Os algoritmos são extremamente necessários para a programação, pois são a base para a implementação de soluções em formato de código.

Entenda Melhor

Algoritmos são como receitas de cozinha para computadores: são conjuntos precisos de passos que guiam como realizar tarefas.

Funcionamento

Quando um algoritmo é executado, significa que estamos seguindo um conjunto de instruções precisas para realizar uma tarefa específica. Cada instrução é seguida exatamente na ordem em que é escrita, sem pular etapas.

Imagine que você está seguindo um manual de instruções para montar um móvel. Você começa com o passo um, depois passa para o dois, e assim por diante, até concluir a montagem. Da mesma forma, um algoritmo segue uma sequência de passos até atingir seu objetivo.

Lógica de Programação

É como pensamos para resolver um problema. Imagine isso como seguir uma receita de bolo. Você precisa entender cada passo e organizá-los de forma lógica para que o bolo saia certo no final. Isso envolve saber usar os ingredientes (dados) da maneira certa e seguir a ordem correta dos passos.

Ou seja, um algoritmo é a sequência de passos pra resolver um problema, enquanto a Lógica de Programação se refere a habilidade de pensar de forma estruturada para desenvolver os algoritmos

Variável

Imagine uma variável como uma caixa onde você pode guardar alguma coisa, como um número ou uma palavra. Você pode dar um nome para essa caixa e colocar diferentes coisas dentro dela enquanto o programa está sendo executado. Essas "caixas" são úteis para armazenar informações temporariamente e usar mais tarde no programa.

Tipos de Variáveis

Em Pascal, as variáveis podem ser classificadas de acordo com o tipo de dados que armazenam. Aqui estão as principais classificações:

  • Numéricas
    • Inteiras

      • Armazenam números inteiros, como 1, -5, 100.

    • Reais

      • Armazenam números com parte fracionária, como 3.14, -0.5, 10.75.

  • Lógicas

    • Armazenam valores lógicos, como verdadeiro (true) ou falso (false). São frequentemente usadas em expressões booleanas.

  • Caractere

    • Armazenam caracteres individuais, como letras, números ou símbolos. Por exemplo, 'A', 'b', '7', '@'.

  • String

    • Armazenam sequências de caracteres, ou seja, palavras ou frases. Em Pascal, essas variáveis são chamadas de strings.

Pseudocódigo

Escrever um pseudocódigo é como dar instruções para resolver um problema usando uma linguagem simples que todos entendam, sem se preocupar com as regras de programação.

É como fazer um esboço de um plano antes de começar a fazer algo. Você descreve passo a passo o que precisa ser feito de forma clara e direta, ajudando a entender como resolver o problema antes de escrever o código real em uma linguagem de programação específica.

Os algoritmos que vamos trabalhar serão feitos em pseudocódigo.

Regras para Criar um Algoritmo

Usar somente um verbo por frase (Um comando)

O objetivo precisa ser claro

Usar frases curtas e simples

Não utilizar palavras com duplo sentido

Componentes Básicos de um algoritmo

Dados de entrada

São os dados que o usuário vai digitar. São os dados necessários para que o programa seja executado. São variáveis.

Dados de processamento

São aqueles que o algoritmo "manipula" ou "trabalha" durante sua execução para produzir o resultado desejado.

Dados de saída

São os resultados ou informações que o algoritmo produz após processar os dados de entrada. Eles representam a resposta ou o resultado final do algoritmo com base nas operações realizadas.
