Tecnologias Web - 01

FRAMEWORKS

Desenvolvimento Simplificado para Web

Frameworks são ferramentas poderosas que simplificam e aceleram o desenvolvimento web interativo e responsivo. Eles oferecem uma estrutura pré-definida, componentes reutilizáveis e funcionalidades prontas para uso. Em outras palavras, são como conjuntos de ferramentas que permitem aos desenvolvedores criar sites e aplicativos web de forma mais eficiente, concentrando-se no design e na funcionalidade, em vez de lidar com detalhes técnicos complexos.

Neste contexto, é importante ressaltar que os frameworks são essenciais para agilizar o processo de desenvolvimento, já que oferecem soluções pré-construídas para problemas comuns encontrados no desenvolvimento web.

Entenda melhor

Um framework é um código já estruturado, sendo assim, o programador não precisa perder tempo focando na construção do código em si, só na parte do conteúdo, o que acelera bastante o processo

Conceitos relacionados

Bibliotecas

Conjuntos de códigos prontos que realizam tarefas específicas, como jQuery para manipulação de HTML com JavaScript.

APIs (Application Programming Interfaces)

Conjuntos de funções que permitem a comunicação entre diferentes softwares. Por exemplo, a API do Google Maps para integrar mapas em aplicativos.

Módulos

Partes individuais de um programa que podem ser reutilizadas. Em Node.js, por exemplo, módulos permitem a inclusão de funcionalidades específicas.

Componentes

Partes independentes e reutilizáveis da interface de usuário. React, por exemplo, utiliza componentes para construir interfaces.

Bibliotecas

Sem Math.js

                    // Função para calcular a média
function calcularMedia(lista) {
    let soma = 0;
    for (let num of lista) {
        soma += num;
    }
    return soma / lista.length;
}

// Função para calcular o desvio padrão
function calcularDesvioPadrao(lista) {
    let media = calcularMedia(lista);
    let somaDiferencasQuadradas = 0;
    for (let num of lista) {
        somaDiferencasQuadradas += Math.pow(num - media, 2);
    }
    let desvioPadrao = Math.sqrt(somaDiferencasQuadradas / lista.length);
    return desvioPadrao;
}

// Exemplo de uso
let numeros = [5, 7, 2, 10, 8];
let media = calcularMedia(numeros);
let desvioPadrao = calcularDesvioPadrao(numeros);

console.log('Média:', media);
console.log('Desvio Padrão:', desvioPadrao);

Princípios Básicos de Frameworks

Reutilização

Utilizar códigos prontos para evitar reescrever o mesmo código repetidamente.

Modularidade

Dividir um programa em partes menores e independentes que podem ser reutilizadas.

Abstração

Ocultar detalhes complexos do funcionamento interno e expor apenas o necessário.

Extensibilidade

Permitir que novos componentes sejam adicionados sem alterar o código existente.

React / Angular / Vue.js

React

O React é uma ferramenta criada pelo Facebook que ajuda os desenvolvedores a fazer sites mais rápidos e dinâmicos. Ele permite criar pequenas partes da página que podem mudar sozinhas sem precisar recarregar tudo, como quando aparece um novo comentário ou uma nova foto.

Angular

O Angular é uma ferramenta criada pelo Google que ajuda a fazer sites e aplicações web de forma organizada e eficiente. Ele permite criar pequenas partes da página, chamadas componentes, que podem mudar sozinhas sem precisar recarregar tudo. Com Angular, é fácil construir sites rápidos e dinâmicos, e ele é usado por muitas empresas grandes como Google e Microsoft.

Vue.js

Conhecido por sua simplicidade e fácil adoção, o Vue.js tem uma curva de aprendizado mais suave. Seu núcleo menor e sua flexibilidade o tornam uma ótima escolha para projetos de todos os tamanhos, embora sua documentação e ecossistema ainda possam estar em desenvolvimento.

É crucial entender as diferenças entre esses frameworks para escolher o mais adequado para um projeto específico. Além disso, compreender suas comunidades, recursos disponíveis e casos de uso é fundamental para uma escolha informada.

Data Binding e State

Esses conceitos são muito importantes porque estão diretamente ligados às ferramentas que usamos para fazer sites e aplicativos. Por exemplo, quando usamos coisas como React, Angular ou Vue.js para criar nossos projetos, esses conceitos nos ajudam a tornar esses projetos mais legais e interativos.

O Data Binding é como uma linha que conecta as informações que estão no nosso programa com o que aparece na tela do computador ou celular. Já Props & States são como ferramentas que nos ajudam a enviar e organizar essas informações entre diferentes partes do nosso projeto. Quando usamos esses conceitos junto com os frameworks, torna mais fácil fazer sites e aplicativos que respondem rápido, funcionam bem e são fáceis de usar.

Data Binding

Estabelece uma conexão entre os dados da aplicação e a interface do usuário, garantindo que as atualizações em um lado sejam refletidas no outro. React, Angular e Vue.js abordam o data binding de maneiras diferentes, com destaque para unidirecional (React), bidirecional (Angular) e bidirecional por padrão (Vue.js).

Imagine que você está usando um aplicativo de lista de tarefas, como um bloco de notas, por exemplo.

Com o Data Binding, quando você adiciona uma nova tarefa na lista no aplicativo, essa tarefa aparece automaticamente na tela, sem que você precise atualizar a página. Isso acontece porque o aplicativo está conectado aos dados (as tarefas que você adiciona) e à interface do usuário (o que você vê na tela). Qualquer mudança que você faça na lista de tarefas é instantaneamente refletida na tela, e vice-versa.

Agora, cada framework (React, Angular e Vue.js) tem sua própria maneira de lidar com essa conexão entre dados e interface do usuário. No React, por exemplo, essa conexão é unidirecional, o que significa que as mudanças nos dados atualizam a interface do usuário, mas não o contrário. No Angular, é bidirecional, então as mudanças nos dados também atualizam a interface do usuário, e vice-versa. E no Vue.js, essa conexão é bidirecional por padrão, o que significa que as mudanças nos dados e na interface do usuário são sincronizadas automaticamente.

Props e State

Componentes fundamentais para construir aplicações interativas e dinâmicas. O state representa os dados mutáveis de um componente, enquanto as props são dados imutáveis passados de um componente pai para um filho, permitindo a comunicação entre eles.

Imagine que você tem um aplicativo de lista de tarefas no seu computador.

Os Props seriam como os detalhes de cada tarefa na sua lista, como o título, a descrição e se está concluída ou não. Por exemplo, para cada tarefa na lista, você precisa dessas informações para saber o que fazer.

Os States seriam como o estado geral do seu aplicativo de lista de tarefas. Por exemplo, se você está visualizando todas as tarefas, apenas as concluídas ou apenas as pendentes. Esse estado pode mudar quando você clica em botões ou faz outras ações no aplicativo.

Então, Props são detalhes específicos de cada parte do seu aplicativo, enquanto States são o estado geral do seu aplicativo como um todo. Esses conceitos nos ajudam a criar aplicativos que funcionam bem e respondem às nossas interações.

Essenciais Front-End

Os frameworks fornecem uma base sólida e reutilizável para o desenvolvimento, o que torna o processo mais organizado e menos complicado. Por exemplo, ao usar frameworks como React, Vue.js ou Angular, os desenvolvedores podem aproveitar recursos como JSX, Virtual DOM, Vuex e diretivas ngModel para simplificar o desenvolvimento front-end. Essas ferramentas permitem criar interfaces de usuário interativas, gerenciar o estado da aplicação de forma eficiente e garantir uma experiência de desenvolvimento mais fluida e produtiva.

Sintaxe JSX

No React, existe uma forma especial de escrever código chamada JSX. É como uma linguagem que mistura JavaScript com HTML. Com o JSX, podemos criar componentes React de uma maneira mais fácil e compreensível. Por exemplo, em vez de escrever algo complicado como React.createElement('div', null, 'Hello World'), podemos escrever simplesmente <div>Hello World</div>. Isso deixa o código mais fácil de entender, especialmente para quem está acostumado com HTML, porque se parece muito com ele.

Virtual DOM

No React, há algo chamado Virtual DOM, que é uma representação da estrutura da página de forma virtual. Quando algo na aplicação muda, o React compara esse Virtual DOM com o DOM real da página. Ele só atualiza as partes que mudaram, em vez de refazer toda a página. Isso faz com que as atualizações na interface do usuário sejam mais rápidas e eficientes.

Vuex

No Vue.js, temos o Vuex, que é como um armazém central para guardar informações importantes da nossa aplicação. Ele nos ajuda a manter tudo organizado e a compartilhar essas informações entre diferentes partes da aplicação. O Vuex usa algumas palavras específicas, como store, state, mutations, actions e getters, para nos ajudar a lidar com essas informações de forma eficiente. Isso é especialmente útil em projetos maiores e mais complicados, onde precisamos ter controle sobre o estado da aplicação.

Digamos que nossa loja tenha uma página de carrinho de compras, onde os usuários podem adicionar produtos ao carrinho e ver o total. Agora, se quisermos garantir que todos os componentes da nossa aplicação tenham acesso ao mesmo carrinho de compras e ao total atual, podemos usar o Vuex.

Então, o Vuex seria como uma caixa grande onde guardamos todas as informações sobre o carrinho de compras, como os produtos que estão nele e o total a ser pago. Cada vez que um usuário adiciona um produto ao carrinho, em vez de ter que passar essa informação de um componente para outro (o que poderia ser complicado em uma aplicação grande), podemos simplesmente colocar essa informação na caixa do Vuex. Todos os componentes da nossa aplicação podem então acessar essa mesma caixa para ler ou modificar as informações do carrinho de compras.

Two-Way Binding

Two-Way Binding é uma técnica de data binding que permite sincronizar automaticamente os dados do modelo com a interface do usuário e vice-versa. Ou seja, quando os dados do modelo são alterados, a interface do usuário é atualizada automaticamente, e quando a interface do usuário é alterada (por exemplo, por meio de um formulário), os dados do modelo são atualizados automaticamente. Isso torna a criação de interfaces interativas e responsivas mais simples e intuitiva.

Imagine que você está preenchendo um formulário em uma página da web. Com Two-Way Binding, sempre que você digita alguma informação no formulário, essa informação é automaticamente atualizada em um modelo de dados por trás dos bastidores. E, ao mesmo tempo, se você mudar algo nesse modelo de dados, a interface do usuário (como o formulário que você está preenchendo) será atualizada para refletir essa mudança.

É como se houvesse uma conexão instantânea e bidirecional entre o que você vê na tela e o que está acontecendo nos dados. Isso torna muito mais fácil construir interfaces onde você pode interagir de forma natural, sem se preocupar em manter os dados e a interface sincronizados manualmente.

React Router

React Router é uma biblioteca de roteamento para aplicações React. Ela permite definir rotas na aplicação e associar cada rota a um componente específico. Isso facilita a navegação entre diferentes partes da aplicação, tornando-a mais organizada e fácil de usar. Com o React Router, é possível criar aplicações de página única (SPAs) com navegação fluida e intuitiva.

Imagine que você está navegando em um site com várias páginas. Com o React Router, é como se essas diferentes páginas fossem componentes que você pode trocar facilmente. Você define rotas para cada "página" da sua aplicação e associa essas rotas aos componentes correspondentes. Então, quando você clica em um link para ir para uma nova "página", na verdade você está apenas trocando o componente que está sendo exibido na tela.

Isso faz com que a navegação na sua aplicação seja muito mais suave e organizada. Em vez de carregar uma página completamente nova a cada clique, você está apenas atualizando partes específicas da sua aplicação, criando uma experiência de usuário mais rápida e intuitiva. E o melhor é que tudo isso acontece dentro de uma única página web, tornando sua aplicação uma SPA (Single Page Application).

Diretiva ngModel

A diretiva ngModel é uma ferramenta muito usada no Angular. Ela faz uma conexão especial entre as informações que você coloca nos formulários e os dados da sua aplicação. Isso significa que, se você mudar algo em um formulário, essa mudança será automaticamente refletida nos dados da sua aplicação, e vice-versa. Com isso, criar formulários no Angular fica muito mais fácil e seu código fica mais organizado e fácil de entender.

Vamos imaginar que estamos criando um formulário de cadastro simples em uma aplicação Angular, e queremos usar a diretiva ngModel para fazer a conexão entre os campos do formulário e os dados da nossa aplicação.

Digamos que temos um formulário com campos para nome, e-mail e senha. Usando a diretiva ngModel, podemos criar uma ligação direta entre esses campos do formulário e os dados na nossa aplicação.

Então, se um usuário digitar seu nome, e-mail e senha no formulário, essas informações serão automaticamente armazenadas nos dados da nossa aplicação, graças à ngModel. E se decidirmos mudar algum desses dados diretamente no código, por exemplo, alterar o e-mail, essa mudança será refletida instantaneamente nos campos do formulário, sem que precisemos fazer nada extra.

Isso torna o processo de criação e gerenciamento de formulários muito mais simples e intuitivo, tanto para quem está desenvolvendo a aplicação quanto para quem está usando.

TypeScript

TypeScript é um superconjunto tipado do JavaScript que adiciona tipagem estática opcional ao JavaScript. Ele permite aos desenvolvedores escrever código JavaScript mais seguro e escalável, detectando erros em tempo de compilação e fornecendo ferramentas avançadas de autocompletar e refatoração. O TypeScript é amplamente utilizado em projetos Angular, mas também pode ser adotado em projetos React e Vue.js para melhorar a qualidade e a manutenibilidade do código.

Imagine o TypeScript como uma versão melhorada do JavaScript, onde você pode especificar o tipo de cada dado que está usando. Por exemplo, você pode dizer que uma variável é do tipo número, texto ou qualquer outro tipo de dado. Isso ajuda a evitar erros no seu código, porque o TypeScript verifica se os tipos de dados estão sendo usados corretamente.

Então, quando você está escrevendo seu código em TypeScript, ele te avisa imediatamente se você tentar fazer algo que não está de acordo com os tipos de dados que você especificou. Isso é ótimo porque você pode detectar e corrigir erros antes mesmo de executar o código.

Além disso, o TypeScript também oferece outras ferramentas úteis, como autocompletar e refatoração, que tornam o processo de escrever código mais rápido e eficiente.

Ele é muito usado em projetos Angular, mas também pode ser usado em projetos React e Vue.js para melhorar a qualidade e a facilidade de manutenção do código.
